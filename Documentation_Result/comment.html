Commentaire¶

Il est très important que tu commentes ton code. Tu n’es pas obligé de faire de la documentation avec les normes (même si je te le conseille) mais tu dois au minimum commenter tes boucles for/while, tes if ou même tes lignes de code compliquées au cas ou quelqu’un d’autre relise ton code ou même si toi tu veux revenir dessus plus tard.

Plusieurs types de commentaire¶

Il est important de bien commenter sa page python. En Python il existe 2 types de commentaire différent :

  • les docstrings qui démarrent par le caractère “”” ou ‘’‘
    """Je suis une docstring"""
'''Je suis une autre manière d'écrire les docstrings'''
  • les commentaires classiques qui démarrent par le caractère #
    #Je suis un commentaire classique

Les docstring sont faits pour écrire des longs commentaires et les commentaires classiques pour ajouter une précision sur le code.

Normes pour commenter¶

En Python un outil très pratique permet de faire de la documentation automatiquement, le module Sphinx. Il a d’ailleurs été utilisé pour écrire cette documentation et beaucoup de documentation officielle que l’on trouve sur internet. Pour créer une documentation Sphinx lit les docstrings présentes dans le code Python. Il faut donc qu’elles soient écrites avec une norme et une structure particulière.

En Python il existe 2 normes :

  • La norme Numpy docstring
  • La norme Google docstring
